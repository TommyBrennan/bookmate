### Issue #96: Code review round 11 (PR #97 — merged + deployed)

3 bugs fixed:

1. **Stale `max_group_size` in approve transaction** (CRITICAL) — Applications route read `max_group_size` outside the transaction closure but used it inside. A concurrent PATCH to the listing could change the value between the outer read and the transaction execution, allowing one extra member past the cap. Fixed by re-reading `max_group_size` inside the transaction, matching the pattern already used in the join route.

2. **Missing `useEffect` cleanup on HomePage** (IMPORTANT) — `debounceRef` and `abortRef` were not cleaned up on unmount. Navigating away during a debounced search left orphaned timers and in-flight fetches that could call setState after unmount. Added cleanup effect to clear timeout and abort controller.

3. **Duplicate author notifications on group fill** (IMPORTANT) — When the last person joined, the author received both a `new_member` notification and a `group_full` notification. Now skips the `new_member` notification when the group fills, since `group_full` is more informative and already implies the join event.
