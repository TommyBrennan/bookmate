### Issue #80: Code review round 4 (PR #81 — merged & deployed)

6 bugs fixed:

1. **Email duplicate check not normalized** — Registration used raw email for duplicate SELECT but stored lowercase. Case-variant emails bypass 409 and hit 500 from UNIQUE constraint. Fixed by normalizing before the check.

2. **Application status hidden when requires_approval off** — GET listing only showed pending application status when `requires_approval` was true. If toggled off after someone applied, the UI showed a "Join" button instead of pending state. Removed the `requires_approval` guard.

3. **Join without checking pending applications** — Users with pending applications could directly join if `requires_approval` was toggled off. Added pending application check to join route.

4. **Rate limiter setInterval leak** — `setInterval` cleanup ran on every module import during hot reload, creating duplicate intervals. Guarded with global singleton pattern.

5. **Platform link exposure to non-members** — `telegram_link` and `discord_link` returned in listing response to all visitors. Now stripped for non-members/non-authors.

6. **Date filter double-apply** — Two conflicting `start_date >= ?` conditions when both `includePast=false` and `start_date_from` were set. Merged into single effective condition using the more restrictive date.

**Testing:** 128 tests pass (2 new for pending application guard), ESLint clean, build passes. Deployed and verified healthy.
