### Issue #108: Code review round 17 (PR #109 — merged + deployed)

4 fixes:

1. **`session.destroy()` not awaited** (IMPORTANT) — Deleted user's cookie never actually cleared on client side because `destroy()` returns a Promise that wasn't awaited. Fixed with `await`.

2. **PATCH /api/notifications crashes on missing body** (IMPORTANT) — `req.json()` threw unhandled SyntaxError when body is missing/malformed, producing 500 instead of 400. Fixed with try/catch wrapper. Added test.

3. **BookSearch abort leak on early return** (IMPORTANT) — In-flight fetch not aborted when query shortened below 3 chars, causing potential state update on unmounted component. Fixed by aborting on early return path.

4. **Middleware blocks server action probes** (HARDENING) — Bots sending `Next-Action: x` header caused repeated PM2 error logs. App uses zero server actions, so added middleware to return 400. Verified working in production.
