### Issue #104: Code review round 15 (PR #105 — merged + deployed)

6 fixes:

1. **`auto-discord` uses `getSession()` instead of `requireAuth()`** (IMPORTANT) — Bypassed DB user-existence check that all other authenticated endpoints perform. Deleted users with valid cookies could still access this endpoint. Fixed by replacing with `requireAuth()`.

2. **LIKE wildcard metacharacters not escaped in search inputs** (IMPORTANT) — `%` and `_` in search terms acted as SQL LIKE wildcards. Searching for `%` matched every listing. Fixed with `escapeLike()` function and `ESCAPE '\'` clause on all LIKE conditions.

3. **Mark-all-notifications-read UPDATE unbounded** (IMPORTANT) — Bulk UPDATE with no LIMIT could lock SQLite WAL on large notification sets. Fixed with subquery LIMIT 1000 to bound write duration.

4. **Profile reading history query unbounded** (IMPORTANT) — No LIMIT on readings query. Added LIMIT 200 to prevent memory/bandwidth issues.

5. **ORDER BY via string interpolation** (HARDENING) — Replaced if/else chain with explicit `ORDER_MAP` allowlist for sort parameter. Safe today but fragile pattern.

6. **Nodemailer transporter cached permanently** (IMPORTANT) — Transporter created once and never rebuilt even if SMTP env vars changed. Added config hash comparison to invalidate and rebuild when env vars change.

All 201 tests pass, ESLint clean, build clean, CI green.
