### Issue #98: Code review round 12 (PR #99 — merged + deployed)

3 fixes:

1. **PATCH race condition on `is_full` reopen** (CRITICAL) — When increasing `maxGroupSize` to reopen a full listing, the member count read and `is_full` check happened outside any transaction. A concurrent join could complete between these reads and the UPDATE, incorrectly clearing `is_full` and allowing extra members beyond the cap. Fixed by wrapping the entire PATCH update logic in a `db.transaction()` that re-reads member count and `is_full` inside the transaction.

2. **Internal platform IDs leaked in GET response** (IMPORTANT) — `telegram_chat_id` and `discord_channel_id` were included in the listing GET response for all users. These are internal bot/channel identifiers not meant for client exposure. Fixed by stripping them from the response regardless of auth status.

3. **Misleading `safeCompare` comment** (MINOR) — Comment claimed length-timing was fully masked by self-comparison, which is not accurate. Updated to be truthful about the limitation.
