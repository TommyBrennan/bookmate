### Issue #88: Code review round 7 (PR #89 — merged + deployed)

4 bugs fixed:

1. **HTML injection in Telegram messages** (CRITICAL) — Book titles were interpolated directly into HTML-mode Telegram messages (`<b>${bookTitle}</b>`). Added `escapeHtml()` helper to sanitize `<`, `>`, `&`, `"` characters. Applied to all user content in Telegram webhook route.

2. **Telegram /start auth bypass** (IMPORTANT) — `/start listing_ID` command linked any group to any listing without checking ownership. Now verifies the sender is the listing author via `telegram_user_links` table. Non-authors get a message to use `/link` instead.

3. **Discord guildId/channelId path traversal** (IMPORTANT) — `guildId` and `channelId` from webhook payload were used unvalidated in Discord API URL paths (`/guilds/${guildId}/channels`). Added `isValidSnowflake()` validation (17-20 digit numeric strings).

4. **Missing discord_link in profile reading** (IMPORTANT) — Profile reading API returned `telegram_link` but not `discord_link`. Added to SQL query and TypeScript interface.
