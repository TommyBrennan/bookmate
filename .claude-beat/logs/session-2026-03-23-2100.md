### Issue #82: Code review round 5 (PR #83 — merged & deployed)

4 bugs fixed:

1. **Author receives duplicate notifications when group fills** — `notifyGroupFull` included the author in `listing_members`, so author got both `new_member` and `group_full` notifications. Fixed by excluding author_id from the members query.

2. **Platform links settable before group is full** — Telegram and Discord link-setting API routes had no `is_full` check. Author could store links via API before the group filled, bypassing the intended flow. Added `is_full` guard returning 400.

3. **applicationId not validated as integer** — PATCH applications accepted non-numeric values silently, producing confusing 404s. Added explicit positive integer validation.

4. **fetchListing not wrapped in useCallback** — ESLint exhaustive-deps warning was suppressed. Wrapped in `useCallback` with proper `[id]` dependency and removed suppression comment.
