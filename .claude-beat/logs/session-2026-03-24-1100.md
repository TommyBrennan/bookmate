### Issue #102: Code review round 14 (PR #103 — merged + deployed)

7 fixes:

1. **BOT_TOKEN/CLIENT_ID frozen at module load time** (CRITICAL) — `telegram.ts`, `discord.ts` captured env vars into module-level constants at import time. If vars weren't set at that moment, `isBotConfigured()` permanently returned false and API calls used empty tokens. Fixed by reading `process.env` live inside helper functions.

2. **`isConfigured` frozen at module load in email.ts** (IMPORTANT) — Same pattern as above for SMTP config. Fixed with `checkConfigured()` function that reads `process.env` on each call.

3. **`requireAuth` never verified user existence** (CRITICAL) — Only checked `session.userId` was truthy in the cookie. A deleted user retained full access. Fixed by adding a DB existence check and auto-syncing stale `displayName` from the cookie.

4. **`notifyGroupFull` re-queried `platform_preference` outside transaction** (IMPORTANT) — Between join transaction commit and this query, a concurrent PATCH could change the platform. Fixed by passing `bookTitle` and `platformPreference` as parameters from the caller (captured before/inside the transaction).

5. **PATCH listing TOCTOU on platform link guard** (IMPORTANT) — The check preventing edits after a Telegram/Discord link is shared was outside the transaction. A concurrent POST could set the link in between. Fixed by re-checking inside the `updateTransaction`.

6. **Profile PATCH missing type validation** (IMPORTANT) — `displayName` and `bio` were `.trim()`-ed without `typeof` check, causing unhandled TypeError on non-string input. Fixed with type guards.

7. **Test updated** — `notifyGroupFull` mock expectation updated for new 3-param signature.

All 201 tests pass, ESLint clean, build clean, CI green.
