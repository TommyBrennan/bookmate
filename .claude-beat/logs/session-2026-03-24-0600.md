### Issue #92: Code review round 9 — client-side bugs (PR #93 — merged + deployed)

7 bugs fixed:

1. **Race condition in `fetchListings`** (CRITICAL) — Filter/pagination changes fired concurrent fetches with no cancellation. Stale slower responses could overwrite newer results. Added `AbortController` to cancel in-flight requests, matching the pattern already used in `BookSearch`.

2. **`start_date` off-by-one in negative UTC timezones** (IMPORTANT) — `new Date("2026-04-01")` is parsed as UTC midnight, which displays as March 31 in timezones west of UTC. Fixed by appending `T00:00:00` for local time interpretation on all display paths (page.tsx, ListingDetail.tsx, profile/page.tsx).

3. **Raw `<Image>` without `unoptimized` in create/edit pages** (IMPORTANT) — Inconsistent with `BookCover` component which uses `unoptimized`. Could cause runtime errors if cover URL from unexpected domain. Added `unoptimized` prop.

4. **Notifications page: no error handling on fetch** (IMPORTANT) — Missing `.catch()` on outer fetch caused infinite loading spinner on network failure. Added error state, error UI with retry button, and silent `.catch()` on mark-as-read PATCH.

5. **`<button role="option">` invalid ARIA in BookSearch** (IMPORTANT) — Interactive element as listbox option breaks screen reader combobox pattern. Changed to `<div role="option">`.

6. **Edit page form shown with default values on fetch error** (IMPORTANT) — When listing fetch failed, form rendered with hardcoded defaults (4 readers, text format) and was submittable. Now shows error message + back button instead.

7. **Profile auth guard: network error leaves blank form** (MODERATE) — Network failure in `/api/auth/me` call left user on empty profile form. Now redirects to login on network error.
